# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dark mode toggle to the home header, with full dark styling across header, quote card, favorites drawer, daily badge, and footer, defaulting to the OS preference and persisting the user's choice.

**Architecture:** Use `next-themes` for theme state (class strategy on `<html>`, matches existing `darkMode: 'class'` Tailwind config), a small `ThemeToggle` client component for the switch, and a Tailwind `dark:` utility pass across the existing components. Remove unused `--background`/`--foreground` CSS variables from `globals.css` as part of the cleanup.

**Tech Stack:** Next.js 14 (App Router), Tailwind CSS, next-themes, lucide-react.

> Note on testing: this frontend has no component/unit test framework configured (no jest/vitest, no existing `*.test.tsx` files). Each task below verifies with `npm run type-check` (already in `package.json`). The final task is a comprehensive manual browser verification, matching the "Teste manual" section of the design spec.

### Task 4: Create ThemeToggle component

**Files:**
- Create: `frontend/src/components/ThemeToggle.tsx`

- [ ] **Step 1: Create the component**

Create `frontend/src/components/ThemeToggle.tsx`:

```tsx
'use client';

import { useEffect, useState } from 'react';
import { Moon, Sun } from 'lucide-react';
import { useTheme } from 'next-themes';

export function ThemeToggle() {
  const { resolvedTheme, setTheme } = useTheme();
  const [mounted, setMounted] = useState(false);

  useEffect(() => {
    setMounted(true);
  }, []);

  if (!mounted) {
    return <div className="w-[14px] h-[14px]" aria-hidden="true" />;
  }

  const isDark = resolvedTheme === 'dark';

  return (
    <button
      onClick={() => setTheme(isDark ? 'light' : 'dark')}
      title={isDark ? 'Ativar tema claro' : 'Ativar tema escuro'}
      className="flex items-center text-stone-400 hover:text-stone-600
                 dark:text-stone-500 dark:hover:text-stone-300 transition-colors duration-200"
    >
      {isDark ? <Sun size={14} /> : <Moon size={14} />}
    </button>
  );
}
```

The `mounted` guard avoids a hydration mismatch: `useTheme()` returns `undefined` for `resolvedTheme` during SSR, so the icon is only rendered once mounted on the client. The placeholder `div` matches the icon's footprint (14x14) so the header doesn't shift when it appears.
